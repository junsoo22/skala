# SKALA 스터디 플래너

## 1. 구현한 기능
- F1 학습 목표 추가 (빈 값 · 공백만 입력 시 거부)
- F2 완료 토글 · 삭제 (이벤트 위임, 목록에 리스너 1개)
- F3 필터(전체/진행중/완료) 및 진행률 표시
- F4 localStorage 저장 — 새로고침해도 목록 유지
- F5 마감일 입력 및 지난 목표 시각적 강조
- F6 목표 검색 (실시간 반영)
- F7 분류(HTML/CSS/JS)별 남은 개수 요약

### 도전 과제
- 목표 제목 더블클릭 수정 (Enter 저장 / Escape 취소)
- 정렬(기본/마감임박순/최신순)
- 다크 모드 토글 (CSS 변수 교체, localStorage에 기억)
- 상황별 빈 화면 안내 문구 (전체 비었을 때 / 필터로 비었을 때 / 검색으로 비었을 때 각각 다르게 표시)
- 목표 추가·삭제 시 부드러운 전환 애니메이션 (`prefers-reduced-motion` 존중)

### 차별화 포인트 — 학습 활동 히트맵 · 연속 학습일
목표를 완료한 날짜를 기록해두었다가, GitHub 잔디(contribution graph)처럼 최근 12주를 CSS Grid로 그리고, 오늘부터 며칠 연속으로 목표를 완료했는지("스트릭")를 계산해 보여준다. 요구사항에는 없지만 코딩 학습 습관을 시각적으로 보여주는 게 스터디 플래너의 취지에 맞다고 생각해 추가했다.

### 차별화 포인트 — 경험치(XP) · 레벨 시스템
목표를 완료할 때마다 XP +10을 얻고, 50XP마다 레벨이 오른다. 공부를 게임처럼 만들어 동기부여를 주고 싶어서 추가했다. 레벨이 오르는 순간에는 레벨 숫자가 살짝 튀어오르는 연출이 들어간다(`prefers-reduced-motion` 존중). XP는 목표를 나중에 지우거나 완료를 취소해도 깎이지 않는 누적 기록으로 저장해서, "한 번 완료했던 사실"이 사라지지 않게 했다.

## 2. 실행 방법
1. VS Code에서 이 폴더(`이름_스터디플래너`)를 연다 — `planner`가 아니라 이 폴더 자체를 열어야 `css/style.css` 등 상대 경로가 정상 동작한다.
2. 오른쪽 아래 "Go Live" 또는 우클릭 → "Open with Live Server"로 `index.html`을 연다.
3. `file://`로 더블클릭해서 열면 오늘의 팁(`fetch`)이 동작하지 않으므로 반드시 Live Server를 사용한다.

## 3. 어려웠던 점과 해결
- **이벤트 위임 개념**: 처음에는 목록 항목 하나하나에 클릭 리스너를 달아야 하는 줄 알았는데, 그렇게 하면 나중에 추가되는 항목에는 리스너가 안 걸린다는 문제가 있었다. `listEl`(부모) 하나에만 리스너를 걸고, `event.target.closest(".item")`으로 실제 클릭된 항목을, `event.target.matches(".item-check")` / `.item-del`로 어떤 버튼이 눌렸는지 구분하는 방식으로 해결했다. 항목이 몇 개든 리스너는 하나뿐이라 새로고침 없이 추가/삭제가 반복돼도 항상 동작한다.
- **XSS 방지 (escapeHtml)**: 목표 제목을 `innerHTML`로 화면에 그리다 보니, 사용자가 `<script>`나 `<img onerror=...>` 같은 걸 입력하면 그대로 실행될 수 있다는 걸 알게 됐다. `escapeHtml()` 함수로 `div.textContent`에 넣었다가 다시 `innerHTML`을 꺼내는 방식으로 태그를 문자 그대로 이스케이프한 뒤에만 템플릿에 넣도록 고쳤다.
- **필터 + 검색 동시 적용(F3 + F6)**: 필터 탭(전체/진행중/완료)과 검색어를 각각 따로 처리하면 서로 덮어써서 하나만 동작하는 문제가 있었다. `visible()` 함수 안에서 필터 조건을 먼저 걸러낸 결과에 검색어 조건을 이어서(체이닝) 적용하도록 정리해서, 두 조건이 항상 함께 반영되도록 만들었다.
- **HTML 기본 툴팁(title 속성)이 너무 느림**: 학습 활동 히트맵 칸에 마우스를 올리면 날짜가 뜨게 만들 때 처음엔 `title` 속성을 썼는데, 브라우저가 자체적으로 정한 지연 시간(약 1초) 때문에 마우스를 올려도 한참 있어야 날짜가 보였다. 이 지연 시간은 HTML/CSS로 줄일 수 없는 브라우저 기본 동작이라, 아예 `title` 대신 `data-tooltip` 커스텀 속성에 날짜를 담고 CSS `::after`로 `content: attr(data-tooltip)`을 그려서 직접 툴팁을 구현했다. 평소엔 `opacity: 0`으로 숨겨두고 `:hover`일 때 `opacity: 1`로 바꾸면서 `transition-delay`를 0.05초로 짧게 줘서, 사실상 즉시 뜨는 툴팁을 만들었다. 그 과정에서 히트맵을 감싼 `overflow-x: auto`가 툴팁을 잘라 보이게 하는 부작용도 발견했는데, 히트맵이 항상 177px(12주×12px) 고정 폭이라 애초에 가로 스크롤이 필요 없다는 걸 깨닫고 그 속성 자체를 지워서 해결했다.

## 4. 미완성 항목
없음 — F1~F7 필수 요구사항과 도전 과제 5개(수정 기능, 정렬, 다크 모드, 빈 화면 안내, 애니메이션)를 모두 구현했다.
